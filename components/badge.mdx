Use the `Badge` component to attach concise status labels or category tags to any piece of content. Badges render as a `<span>`, `<button>`, or `<a>` element depending on the props you provide — making them equally useful for static labels, clickable filters, and navigational chips.

## Installation

```bash
npm install @mintlify/components
```

## Import

```tsx
import { Badge } from '@mintlify/components';
```

## Basic Usage

```tsx
<Badge color="green" shape="pill" size="sm">New</Badge>
```

<Badge color="green" shape="pill" size="sm">New</Badge>

## Props

<ParamField path="children" type="ReactNode" required>
  The text or elements to render inside the badge.
</ParamField>

<ParamField path="color" type="string">
  Controls the badge color. Accepts: `gray`, `blue`, `green`, `orange`, `yellow`, `red`, `purple`, `white`, `surface`, `white-destructive`, `surface-destructive`.
  Defaults to `gray`.
</ParamField>

<ParamField path="shape" type="'rounded' | 'pill'">
  Sets the border-radius style. `rounded` gives a subtle corner radius; `pill` creates a fully rounded capsule shape. Defaults to `rounded`.
</ParamField>

<ParamField path="size" type="'xs' | 'sm' | 'md' | 'lg'">
  Controls the overall size (padding and font size). Defaults to `md`.
</ParamField>

<ParamField path="variant" type="'solid' | 'outline'">
  `solid` fills the badge with the chosen color; `outline` shows a colored border with a transparent background. Defaults to `solid`.
</ParamField>

<ParamField path="disabled" type="boolean">
  Reduces opacity and disables pointer events. When set, the badge always renders as a `<span>` regardless of other props.
</ParamField>

<ParamField path="href" type="string">
  When provided, the badge renders as an `<a>` element. Takes precedence over `onClick` for element selection.
</ParamField>

<ParamField path="onClick" type="() => void">
  When provided (and no `href`), the badge renders as a `<button>` element.
</ParamField>

<ParamField path="leadIcon" type="ReactNode">
  An icon to render before the children. Pass a string to use a named icon from the configured icon library, or pass a React element for a custom inline icon.
</ParamField>

<ParamField path="tailIcon" type="ReactNode">
  An icon to render after the children. Pass a string to use a named icon from the configured icon library, or pass a React element for a custom inline icon.
</ParamField>

<ParamField path="iconType" type="'brands' | 'duotone' | 'light' | 'regular' | 'sharp-duotone-solid' | 'sharp-light' | 'sharp-regular' | 'sharp-solid' | 'sharp-thin' | 'solid' | 'thin'">
  The style variant of the icon when `leadIcon` or `tailIcon` is a string name. Defaults to `regular`.
</ParamField>

<ParamField path="iconLibrary" type="'fontawesome' | 'lucide'">
  The icon library to source icons from when `leadIcon` or `tailIcon` is a string name. Defaults to `fontawesome`.
</ParamField>

<ParamField path="className" type="string">
  Additional CSS class names to merge onto the badge element.
</ParamField>

## Examples

### Colors

All available color values:

```tsx
<Badge color="gray">gray</Badge>
<Badge color="blue">blue</Badge>
<Badge color="green">green</Badge>
<Badge color="orange">orange</Badge>
<Badge color="yellow">yellow</Badge>
<Badge color="red">red</Badge>
<Badge color="purple">purple</Badge>
<Badge color="white">white</Badge>
<Badge color="surface">surface</Badge>
<Badge color="white-destructive">white-destructive</Badge>
<Badge color="surface-destructive">surface-destructive</Badge>
```

### Shapes

```tsx
<Badge color="blue" shape="rounded">Rounded</Badge>
<Badge color="blue" shape="pill">Pill</Badge>
```

### Sizes

```tsx
<Badge color="purple" size="xs">Extra Small</Badge>
<Badge color="purple" size="sm">Small</Badge>
<Badge color="purple" size="md">Medium</Badge>
<Badge color="purple" size="lg">Large</Badge>
```

### Variants

```tsx
<Badge color="green" variant="solid">Solid</Badge>
<Badge color="green" variant="outline">Outline</Badge>
```

### Interactive Badges

Pass an `onClick` handler to create a clickable button badge, or pass an `href` to create a link badge:

```tsx
{/* Button badge */}
<Badge color="blue" onClick={() => alert('clicked!')}>Click me</Badge>

{/* Link badge */}
<Badge color="orange" href="/changelog">Changelog</Badge>
```

<Tip>
  When you provide both `href` and `onClick`, the badge renders as an `<a>` element and the `onClick` is attached to it as well.
</Tip>

### Disabled State

```tsx
<Badge color="green" disabled>Disabled</Badge>
<Badge color="green" variant="outline" disabled>Disabled Outline</Badge>
```

<Note>
  A disabled badge always renders as a `<span>` — not a `<button>` or `<a>` — to prevent interaction regardless of the `href` or `onClick` props.
</Note>

### With Icons

```tsx
<Badge color="blue" leadIcon="star">Featured</Badge>
<Badge color="red" tailIcon="xmark">Remove</Badge>
```

## Accessibility

- When `href` is set, the badge renders as a native `<a>` element, inheriting all browser link semantics.
- When `onClick` is set, the badge renders as a `<button>`, which is keyboard-focusable and activatable via <kbd>Enter</kbd> and <kbd>Space</kbd> by default.
- When `disabled` is set, the badge renders as a `<span>` with `pointer-events: none`, effectively removing it from the tab order and preventing interaction.
- String icons are decorative and are hidden from screen readers automatically.
